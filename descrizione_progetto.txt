PAGE 01 — /auth/login
Nome: Login
Descrizione generale:
Pagina di accesso a Terminia. Layout centrato, sfondo scuro con gradiente, logo Terminia in cima. Nessuna sidebar. Form di login con email e password. Gestione errori inline. Link a registrazione e recupero password.
Layout:
  Pagina full-screen centrata verticalmente e orizzontalmente
  Card bianca/dark centrata, max-width 400px
  Logo Terminia sopra la card
  Nessuna navbar, nessuna sidebar
Componenti UI:
  Input email con icona mail, validazione formato email
  Input password con icona lucchetto, toggle mostra/nascondi password
  Checkbox "Ricordami"
  Button primario full-width "Accedi"
  Link "Password dimenticata?" sotto il bottone
  Divisore "oppure"
  Button secondario "Continua con Google" con logo Google
  Link in fondo "Non hai un account? Registrati"
Comportamento:
  Submit disabilitato se campi vuoti o email non valida
  Loading spinner sul bottone durante la chiamata
  Errore inline sotto il form se credenziali errate: "Email o password non corretti"
  Su successo → redirect a /dashboard
  Se primo accesso (no company configurata) → redirect a /onboarding
Tabelle:
  users — lettura per ottenere company_id e role post-login via Supabase Auth
PAGE 02 — /auth/register
Nome: Registrazione
Descrizione generale:
Wizard a 2 step per creare account e azienda. Step 1 dati personali, Step 2 dati aziendali. Barra progresso in cima. Al termine crea companies + users e redirect a /onboarding.
Layout:
  Stessa struttura di login — card centrata, no sidebar
  Barra progresso con 2 step in cima alla card
  Step indicator: "1. Il tuo account" → "2. La tua azienda"
Step 1 — Dati personali:
  Input Nome completo
  Input Email
  Input Password con requisiti visivi (8+ caratteri, maiuscola, numero)
  Input Conferma password
  Button "Continua →"
Step 2 — Dati aziendali:
  Input Nome azienda (obbligatorio)
  Input Partita IVA con validazione formato IT (11 cifre) — badge verde/rosso inline
  Select Settore (dropdown con settori principali)
  Select Dimensione (micro / small / medium / large)
  Input Città
  Button "← Indietro" + Button "Crea account →"
Comportamento:
  Validazione P.IVA italiana in tempo reale (algoritmo di controllo cifra check)
  Al submit step 2: INSERT companies → INSERT users con role = admin
  Redirect automatico a /onboarding
Tabelle:
  companies — INSERT al completamento step 2
  users — INSERT al completamento step 2
PAGE 03 — /onboarding
Nome: Onboarding Wizard
Descrizione generale:
Wizard guidato a 4 step per completare il profilo aziendale. Necessario per abilitare BandoRadar e il calcolo reliability score. Ogni step è salvabile — l'utente può uscire e riprendere. L'ultimo step è opzionale.
Layout:
  Layout full-page con sidebar sinistra che mostra i 4 step come checklist
  Contenuto principale a destra con form
  Step completati hanno spunta verde nella sidebar
  Bottone "Salta per ora" in alto a destra su ogni step
Step 1 — Profilo aziendale completo:
  Input Codice ATECO con autocomplete (cerca per codice o descrizione)
  Input PEC
  Input Codice SDI (7 caratteri)
  Input Indirizzo completo (via, CAP, città, provincia)
  Input Codice Fiscale aziendale
Step 2 — Profilo operativo (per BandoRadar):
  Input Fatturato annuo (numerico, in €)
  Input Numero dipendenti
  Multi-select Certificazioni possedute (ISO 9001, ISO 27001, SOA, ecc.)
  Multi-select Regioni di operatività (tutte le regioni italiane)
  Toggle Ha esperienza con contratti PA? (sì/no)
  Se sì: Input Valore contratti PA gestiti (€)
Step 3 — Invita il team:
  Lista dinamica di righe: Input email + Select ruolo (admin/manager/viewer)
  Bottone "+ Aggiungi membro"
  Possibile aggiungere fino a 5 membri nel MVP
  Bottone "Invia inviti"
Step 4 — Carica il primo contratto (opzionale):
  Area drag & drop per PDF
  Testo: "Carica un contratto per vedere Terminia in azione"
  Badge "Opzionale — puoi farlo dopo"
  Bottone "Salta e vai alla dashboard"
  Bottone "Carica e analizza"
Tabelle:
  companies — UPDATE con tutti i dati completati
  users — INSERT per ogni membro invitato
  ai_interviews — INSERT se l'utente carica il primo contratto
PAGE 04 — /dashboard
Nome: Home Dashboard
Descrizione generale:
Centro di comando principale. Mostra lo stato dell'azienda in tempo reale. Prima cosa che vede l'utente ogni mattina. Tutto cliccabile — ogni widget porta alla pagina di dettaglio corrispondente.
Layout:
  Sidebar sinistra fissa con navigazione (presente in tutte le pagine /dashboard/*)
  Area principale con:
  KPI bar in cima (5 card orizzontali)
  Grid 2 colonne con 5 widget
  FAB button "+ Carica documento" fisso in basso a destra
Sidebar navigazione (presente in tutte le pagine dashboard):
text
🏠 Dashboard 📄 Contratti 👥 Controparti 👤 Dipendenti 🧾 Fatture 🎯 BandoRadar 🔔 Alert 📝 Documenti 📊 Analytics ⚙️ Impostazioni
  Avatar utente + nome azienda in fondo alla sidebar
  Badge rosso con count su "Alert" se ci sono alert pending critici
KPI Bar (5 card):
 Contratti attivi — numero con icona contratto — link a /contracts?status=active
 Valore portfolio — somma in € contratti attivi — formato abbreviato (es. €124k)
 In scadenza 30gg — numero con badge arancione se > 0 — link a /contracts?expiring=30
 Alert non gestiti — numero con badge rosso se critici — link a /alerts
 Bandi match >80% — numero con badge verde — link a /bandi
Widget 1 — Scadenze critiche (colonna sinistra, grande):
  Titolo "Prossimi 7 giorni"
  Lista max 5 items, ognuno con:
  Icona tipo (contratto/obbligo/fattura/bando)
  Titolo scadenza
  Entità collegata (nome contratto o controparte)
  Countdown in giorni (es. "tra 3 giorni") colorato: rosso <3gg, arancione <7gg
  Link alla scheda
  Se lista vuota: stato empty "Nessuna scadenza critica 🎉"
Widget 2 — Alert attivi (colonna destra, medio):
  Titolo "Alert non gestiti"
  Lista max 5 alert per priorità decrescente
  Per ogni alert: badge priorità colorato + titolo + bottone "Gestisci"
  Link "Vedi tutti →" in fondo
Widget 3 — BandoRadar Top Match (colonna sinistra, medio):
  Titolo "Bandi compatibili"
  3 card bando con: match score badge, titolo, ente, scadenza countdown, bottone "Valuta"
  Link "Vedi tutti i bandi →"
Widget 4 — Attività recente (colonna destra, piccolo):
  Ultimi 5 contratti caricati/modificati
  Per ognuno: titolo + data + status badge
Widget 5 — Cashflow 30gg (colonna intera, piccolo):
  2 numeri affiancati: "Da incassare € X" (verde) e "Da pagare € X" (rosso)
  Link "Vai alle fatture →"
Tabelle:
  v\_dashboard\_kpi — KPI bar
  v\_contracts\_expiring_soon — Widget scadenze
  alerts — Widget alert (top 5 pending order by priority)
  v\_bandi\_top_match — Widget BandoRadar

  invoices — Widget cashflow (SUM amount_gross per status)

PAGE 07 — /dashboard/contracts/new
Nome: Upload AI / Nuovo Contratto
Descrizione generale:
Pagina per aggiungere un nuovo contratto. Due modalità: upload PDF (analisi AI automatica) o creazione manuale. Il flusso AI include un sistema di intervista per disambiguare il documento.
Layout:
  Pagina con due tab in cima: "Carica PDF" e "Crea manuale"
  Contenuto centrato, max-width 700px
Tab "Carica PDF":
  Area drag & drop grande con icona PDF
  Testo: "Trascina il contratto qui o clicca per selezionare"
  Formati accettati: PDF, DOCX
  Dopo upload: barra progresso "Analisi in corso..."
  Risultato analisi AI:
  Classificazione rilevata (es. "Contratto di fornitura servizi") con confidence %
  Se confidence > 85%: mostra classificazione con bottone "Conferma e continua"
  Se confidence < 85%: mostra domande AI per disambiguare
  Intervista AI (se necessaria):
  Domanda in linguaggio naturale (es. "Questo contratto riguarda un fornitore o un dipendente?")
  Risposte come bottoni/radio — non form testuali
  Max 5 domande
  Preview dati estratti prima del salvataggio:
  Titolo rilevato, tipo, controparte rilevata, valore, date
  Ogni campo modificabile inline
  Bottone "Salva e analizza clausole"
Tab "Crea manuale":
  Select Tipo contratto (dropdown con tutti i tipi)
  Radio Attore: Controparte / Dipendente
  Se Controparte: Select searchable da counterparts + bottone "+ Nuova controparte"
  Se Dipendente: Select searchable da employees + bottone "+ Nuovo dipendente"
  Input Titolo contratto
  Input Numero riferimento
  Select Status (default: draft)
  Sezione "Dati economici" (collassabile):
  Input Valore + Select tipo valore
  Input Termini pagamento (giorni)
  Select Frequenza pagamento
  Select Regime IVA + Input aliquota IVA
  Toggle Ritenuta d'acconto + Input aliquota
  Sezione "Date" (collassabile):
  DatePicker inizio, fine, firma, efficacia
  Toggle Rinnovo automatico → se attivo: Input preavviso giorni + Input durata rinnovo
  Sezione "Note e tag" (collassabile)
  Bottone "Salva contratto"
Tabelle:
  ai_interviews — INSERT per ogni sessione upload
  contracts — INSERT finale
  counterparts — INSERT se nuova controparte rilevata
  employees — INSERT se nuovo dipendente rilevato
  clauses — INSERT post-analisi AI
  obligations — INSERT post-analisi AI
  contract_documents — INSERT file originale
